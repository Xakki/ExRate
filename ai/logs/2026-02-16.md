## Implementation Plan
1.  **Enum Updates**: Added `FRANKFURTER`, `BANK_OF_CANADA`, `BINANCE`, `FRED`, `MOEX`, `BANXICO`, `BCB`, and `WORLD_BANK` to `App\Enum\ProviderEnum`.
2.  **DTO Validation Relaxed**: Updated `RateRequest` and `TimeseriesRequest` to allow currency codes from 2 to 10 characters (replacing `#[Assert\Currency]` with `#[Assert\Length]`). This enables support for cryptocurrencies (BTC, ETH, etc.) and longer codes used by some providers.
3.  **Provider Implementations**:
    - `FrankfurterProvider`: EUR-based ECB data. (Verified via integration test).
    - `BankOfCanadaProvider`: CAD-based official rates from Valet API. (Verified via integration test).
    - `BinanceProvider`: USDT-based crypto rates. Implements both Latest (batch) and Historical (loop) fetching. (Verified via integration test).
    - `FredProvider`: USD-based macro data. Maps specific series IDs (DEX...) to currencies. (Ready, requires real API key).
    - `MoexProvider`: RUB-based Moscow Exchange data. Uses ISS API. Support for `wap_rates` (Latest) and board-specific `history` endpoints. (Ready, history data has external delay/restrictions).
    - `BanxicoProvider`: MXN-based official Mexico rates. (Ready, requires real token).
    - `BcbProvider`: BRL-based Central Bank of Brazil rates via OData API. (Verified via integration test).
    - `WorldBankProvider`: USD-based annual average rates. (Ready, but no data for 2026 yet).
4.  **Service Registration**: All new providers registered in `app/config/services.yaml` with unique IDs (25-32).
5.  **Quality Assurance**:
    - Ran `make phpstan` and `make cs-fix` to ensure code quality.
    - Successfully verified integration tests for Frankfurter, Bank of Canada, Binance, and BCB.
6.  **TZ Update**: Updated `new_providers.md` to mark implemented providers.
