# ExRate — Claude project rules

Symfony 8 / PHP 8.5 exchange-rate aggregator (31 providers). Полный workflow — в `AGENTS.md` (импортирован выше). Этот файл — дельта поверх него для Claude Code.

## Команды

**Все операции — через `make`-таргеты.** `make help` — каталог. Если нужного таргета нет — добавь его в `Makefile`, не запускай команду напрямую. Полный референс таргетов — `.ai/ARCHITECTURE.md`.

Ключевые: `make up | down | ps | logs name=php | migrate | test | test-unit | test-functional | phpstan | cs-fix | cs-check | load-rates | queue-run | console cmd="..."`.

## Definition of Done

Перед закрытием задачи с правкой кода: `make cs-fix && make cs-check && make phpstan && make test-unit && make test-functional`. `make test-integration` — **только по явному запросу**.

## Стиль и архитектура (см. подробно `.ai/CODESTYLE.md`, `.ai/ARCHITECTURE.md`)

- API JSON-ответы — `snake_case`.
- `declare(strict_types=1)` везде. DTO вместо массивов на возврате.
- Named-параметры при >2 аргументах.
- Constants типизированные (`public const string X = 'v';`); никаких magic-numbers/strings — в `services.yaml`/`.env`.
- Даты — `\App\Util\Date::createFromFormat`, не `DateTime::createFromFormat`.
- Коды валют — константы из `\App\Util\Currencies` / `\App\Util\CryptoCurrencies`.
- Функциональные тесты мокают cache и messenger; интеграционные — реальный full-flow (sync messenger в test-env).

## Session log

После правки кода — запись в `.ai/logs/YYYY-MM-DD.md` (использовать последний uncommited файл или создать новый): prompt → план → git-summary. Разделять `---`.

## Зоны "не читать" (не индексировать без необходимости)

`app/vendor/`, `app/var/cache/`, `app/var/log/`, `app/public/icons/crypto/` (130+ иконок), `app/public/bundles/`, `app/composer.lock`. Логи смотреть через `make logs name=php`.

## Запреты

- Не запускать `make logs-follow`, `make composer-bash`, `make test-integration` без явного запроса.
- Не править `.env*`, `app/vendor/`, `app/var/`.
- Удаление файла — очистить содержимое и записать `@deleted` (см. CODESTYLE), а не `rm`.

## Указатели

- Архитектура и data-flow — `.ai/ARCHITECTURE.md`
- Кодстиль и тест-стандарты — `.ai/CODESTYLE.md`
- Шаблон ревью — `.ai/REVIEW.md` (когда просят review)
- Полный devops-флоу AGENTS — `AGENTS.md`
- Скиллы: `/add-provider` (новый провайдер), `/release` (релиз)
- Сабагенты: `provider-explorer`, `db-schema-explorer`
